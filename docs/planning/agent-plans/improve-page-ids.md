# Improve Page IDs

## Goal

Replace the current `normalized-prefix + sha1-suffix` scenario labels in `backstop-config.js` with stable, short, project-scoped page IDs generated by a JS port of `Runner::encodeId`.

## Current State

`backstop-config.js` builds scenario labels via `buildScenarioLabel(urlPath)`:

```js
const buildScenarioLabel = (urlPath) => {
  const normalized = normalizeLabelPart(urlPath) || 'root';
  const prefix = normalized.slice(0, MAX_SCENARIO_LABEL_PREFIX);
  const hash = crypto.createHash('sha1').update(urlPath).digest('hex').slice(0, 10);
  return `${prefix}-${hash}`;
};
```

Labels look like: `about-us-a1b2c3d4e5`

## Proposed Changes

### 1. `src/js/backstop-config.js`

**Add `encodeId(value, seed)` function** — a direct JS port of `Runner::encodeId`:

```js
const ENCODE_ALPHABET = 'swxdyktzhgjfblrpmcqvn';

const encodeId = (value, seed = 0) => {
  // Mirror PHP: crc32b-like 32-bit hash via sha1 first 8 hex chars
  const hashHex = crypto.createHash('sha1').update(value).digest('hex').slice(0, 8);
  const hash = parseInt(hashHex, 16) >>> 0; // unsigned 32-bit
  // Mirror PHP: (hash << 16) | (seed & 0xFFFF) as a BigInt to avoid JS precision loss
  let number = (BigInt(hash) << 16n) | BigInt(seed & 0xFFFF);
  const base = BigInt(ENCODE_ALPHABET.length);

  if (number === 0n) return ENCODE_ALPHABET[0];

  let encoded = '';
  while (number > 0n) {
    encoded = ENCODE_ALPHABET[Number(number % base)] + encoded;
    number = number / base;
  }
  return encoded;
};
```

**Derive project seed** from `INVRT_ID`:

```js
const projectSeed = INVRT_ID
  ? parseInt(crypto.createHash('sha1').update(INVRT_ID).digest('hex').slice(0, 8), 16) >>> 0
  : 0;
```

**Replace `buildScenarioLabel`** with `encodeId(urlPath, projectSeed)`.

**Remove** `MAX_SCENARIO_LABEL_PREFIX`, `normalizeLabelPart`, `buildScenarioLabel` (no longer needed).

### 2. `src/core/Runner.php`

Make `encodeId` **public** (currently `private`). This lets tests verify the PHP and JS implementations stay in sync, and clarifies it is an intentional stable API.

No change to the encoding algorithm.

## What does NOT change

- `generateProjectId` stays the same.
- BackstopJS `id` field is still `INVRT_ID`.
- PHP `encodeId` alphabet and formula are unchanged.
- Screenshot filenames will change (they derive from `label`), so old reference screenshots are invalidated for existing projects when they upgrade. This is acceptable for a feature improvement.

## Note on hash function difference

PHP uses `crc32b`; JS uses the first 8 hex chars of sha1. These produce different numeric values for the same input, so PHP and JS IDs will differ for the same URL path. This is intentional — there is no cross-language comparison requirement. The IDs just need to be stable and project-scoped within each run environment.

If full cross-language parity is ever needed, a JS crc32 library can be added.

## Tests

- Update `tests/bats/workflow.bats` to assert that scenario labels in the generated `backstop-config.json` match the `encodeId` pattern (`^[a-z]+$`) rather than the old `prefix-sha1suffix` pattern.
- No PHP unit test changes needed (PHP `encodeId` is unchanged algorithmically; making it `public` is non-breaking).
